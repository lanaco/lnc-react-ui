# Layout

Used to structure the layout of your application, without styles. Considering that, you are in charge of styling all the pieces,
or you can use the predefined components.

## Component overview

Every component is only a semantic container for your content, which is laid out based on flex.

-`Layout` is the top-level wrapper in which you can nest `Header` `Footer` `Content` `Sidebar` or `Layout` itself.<br/> -`Header` is intended to be used at the top of the `Layout`.<br/> -`Sidebar` is intented to be used at either right or left side of `Layout`, and only has basic collapse function.<br/> -`Content` is intented as the central piece of `Layout`.<br/> -`Footer` is intended to be used at the bottom of the `Layout`.

## Properties

<Props components={{ Layout, Sidebar, Content, Header, Footer }} />

## Examples

Most common layout used for applications. Consists of only `Header`, `Content` and `Footer`.<br/>
